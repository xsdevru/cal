# Вложенные структуры храним как JSON-текст в SQLite

Массивы и объекты доменных моделей (`Schedule.availability`/`overrides`,
`EventType.locations`/`bookingFields`, `Booking.attendee`/`guests`/`location`/`metadata`)
хранятся в SQLite как JSON-строки (TEXT), а не нормализуются в отдельные таблицы;
сериализация/десериализация — в `server/src/lib/serialize.ts`. Нормализуем только корневые
сущности (User, Schedule, EventType, Booking) и связи между ними. Выбрали простоту хранения
против запросов по вложенным полям: приложение читает эти структуры всегда целиком (напр.
все блоки расписания при расчёте слотов), а не фильтрует по ним в SQL.

## Consequences

Нельзя фильтровать/джойнить по содержимому JSON-полей средствами БД — если появятся
запросы вида «все брони в такой-то локации», такое поле придётся вынести в колонку/таблицу.
Также в SQLite у Prisma нет enum — `Booking.status` хранится строкой, значения из
`BookingStatus` контролируются на уровне приложения.
