# MVP на одного владельца, без аутентификации

Бэкенд реализован как MVP для единственного владельца: аутентификации нет, админские
ручки (event-types, schedules, bookings) работают от лица «первого пользователя» в БД
(`server/src/lib/owner.ts` → `getOwner()`), которого создаёт сид. Это сознательное сужение
скоупа, чтобы сфокусироваться на доменной логике (расписания, слоты, брони), а не на
управлении пользователями.

## Consequences

Из кода не видно, что авторизация отсутствует намеренно — `getOwner()` молча берёт первого
юзера. Добавление мультипользовательности потребует ввести аутентификацию и скоупить по
`userId` все админские запросы (сейчас скоупинг зашит на одного владельца). Модель данных
уже готова к нескольким владельцам (`User` → `Schedule`/`EventType` по `userId`).
