# 04 — Slots: расчёт свободных слотов

> Статус: реализовано (ретроспективный issue).

## What to build

Единственная нетривиальная логика бэкенда — эндпоинт `/slots` и модуль `server/src/lib/slots.ts`.
Из `Schedule.availability` (регулярные блоки по дням недели, `HH:MM`) + `overrides`
(исключения на дату) + `EventType.lengthInMinutes` режем рабочие окна на слоты шагом
= длительность встречи, переводим локальное время расписания (`timeZone`) в UTC и группируем
по `YYYY-MM-DD`. Вычитание активных броней добавляется в слайсе #05 — здесь слоты считаются
против пустого набора броней. Логика покрыта юнит-тестами (пишутся первыми, по TDD).

## Acceptance criteria

- [ ] `GET /slots?eventTypeId&start&end&timeZone` возвращает `AvailableSlots.slots`,
      сгруппированные по датам.
- [ ] Слоты нарезаются шагом = `lengthInMinutes` в пределах блоков доступности.
- [ ] Исключения на дату переопределяют регулярные блоки (особые часы / закрытый день без слотов).
- [ ] Локальное время расписания корректно конвертируется в UTC.
- [ ] Юнит-тесты покрывают: блоки → слоты, overrides, конверсию TZ, границу суток/DST.
