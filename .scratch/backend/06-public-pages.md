# 06 — Public pages

> Статус: реализовано (ретроспективный issue).

## What to build

Публичная часть, которую видит гость по ссылке. `/{username}` — профиль владельца со списком
**видимых** типов встреч (`hidden: false`); `/{username}/{slug}` — модель `PublicEventPage`
(профиль + тип встречи). Свободные слоты фронт запрашивает отдельно из `/slots`. Важна
изоляция данных: скрытые типы встреч не отдаются, приватные поля `User` (напр. `email`) не
утекают в публичный ответ (`PublicUser`).

## Acceptance criteria

- [ ] `GET /{username}` возвращает профиль и только не-скрытые типы встреч.
- [ ] `GET /{username}/{slug}` возвращает `PublicEventPage` для конкретного типа встречи.
- [ ] Скрытые (`hidden: true`) типы отсутствуют в публичных ответах.
- [ ] Приватные поля `User` не попадают в `PublicUser`.
- [ ] Публичная страница `/:username/:slug` рендерится без админ-меню, слоты подгружаются.
- [ ] Интеграционные тесты и `make verify-contract` без расхождений.
