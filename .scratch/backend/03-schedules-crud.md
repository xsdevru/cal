# 03 — Schedules: CRUD

> Статус: реализовано (ретроспективный issue).

## What to build

Ресурс расписаний доступности с полным CRUD. Расписание — именованный набор правил в
конкретной таймзоне; у владельца одно дефолтное (`isDefault`). Регулярные недельные блоки
(`availability`) и исключения на дату (`overrides`) хранятся как JSON-колонки. Это данные,
на которых слайс #04 считает свободные слоты.

## Acceptance criteria

- [ ] `GET /schedules` и `GET /schedules/{id}` отдают расписания владельца из БД.
- [ ] `POST /schedules` создаёт расписание с `sch_…` id, таймзоной и блоками доступности.
- [ ] `PATCH /schedules/{id}` (merge-patch) обновляет `availability`/`overrides`/метаданные.
- [ ] `DELETE /schedules/{id}` удаляет расписание.
- [ ] Блоки доступности (дни недели + `HH:MM`) и исключения на дату (особые часы / `closed`)
      корректно сериализуются и читаются.
- [ ] Интеграционные тесты и `make verify-contract` без расхождений.
